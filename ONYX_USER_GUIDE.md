## RAG (검색 증강 생성)

RAG는 자신의 문서를 AI 모델과 함께 사용하여 더 정확한 답변을 얻는 기능입니다.

### 🔄 RAG 작동 원리

```
1️⃣ 문서 업로드
   ↓
2️⃣ 문서 인덱싱 (Embedding 생성)
   ↓
3️⃣ 사용자 질문
   ↓
4️⃣ 유사한 문서 검색
   ↓
5️⃣ AI가 검색된 문서 기반으로 답변 생성
```

### 📄 문서 업로드 방법

#### 1단계: 프로젝트 생성

```
Onyx UI → "Projects" → "New Project"
```

- **프로젝트명**: 예) "회사 정책", "기술 문서"
- **설명**: 프로젝트 목적 작성

#### 2단계: 문서 업로드

```
프로젝트 선택 → "Upload" → 파일 선택
```

**지원 포맷:**
- PDF (`.pdf`)
- 텍스트 (`.txt`, `.md`)
- Word (`.docx`)
- 웹페이지 (URL)
- 스프레드시트 (`.csv`, `.xlsx`)

**파일 크기 제한:**
- 기본값: 250MB (설정에서 변경 가능)

#### 3단계: 인덱싱 대기

```
"Status" → "Indexing..." → "Ready"
```

- 업로드 후 문서가 자동으로 인덱싱됨
- 파일 크기에 따라 몇 분이 소요될 수 있음
- "Ready" 상태가 되면 검색 가능

### 🔍 RAG 검색 사용

#### 기본 사용법

```
1. Chat 창에서 질문 입력
2. 프로젝트 선택 (선택사항)
3. 질문 전송

예시:
"회사 정책에서 휴가 일수는?"
"기술 문서의 API 인증 방법은?"
```

#### 검색 결과 활용

**답변에 포함된 정보:**
- ✅ 검색된 문서 인용
- ✅ 출처 표시
- ✅ 신뢰도 점수

**인용 확인:**
```
AI 답변 하단의 "Citations" 또는 "Sources" 클릭
→ 원본 문서 확인 가능
```

### 🗂️ 커넥터: 자동 문서 동기화

**커넥터**란 외부 소스에서 자동으로 문서를 가져오는 기능입니다.

#### 지원하는 커넥터 (50+)

| 카테고리 | 예시 |
|---------|------|
| **클라우드** | Google Drive, Dropbox, OneDrive, S3 |
| **업무** | Slack, Jira, Confluence, Notion |
| **코드** | GitHub, GitLab, Bitbucket |
| **웹** | 웹사이트, RSS 피드, Sitemap |
| **이메일** | Gmail, Outlook |
| **기타** | Salesforce, HubSpot, Linear |

#### 커넥터 설정 방법

```
Admin Panel → "Connectors" → "Add New Connector"
```

**1단계: 소스 선택**
```
Google Drive / Slack / GitHub 등 선택
```

**2단계: 인증**
```
OAuth 로그인 또는 API 키 입력
```

**3단계: 동기화 설정**
```
- 폴더/채널 선택
- 동기화 주기 설정 (일일, 주간, 월간)
- 필터 설정 (특정 파일 타입만 포함)
```

**4단계: 프로젝트에 연결**
```
문서가 자동으로 프로젝트에 추가됨
```

### 💡 RAG 활용 팁

#### 1️⃣ 효과적인 문서 구조

```
좋은 예:
├── 기술문서
│   ├── API_가이드.pdf
│   ├── 아키텍처.md
│   └── 설정.md
└── 정책
    ├── 휴가정책.pdf
    └── 보안정책.pdf

설명이 명확하고 구조화된 문서 추천
```

#### 2️⃣ 최적의 질문 방법

```
❌ 나쁜 예:
"뭐야?"
"알려줘"

✅ 좋은 예:
"API 인증 방법은 무엇인가?"
"회사 휴가 정책에서 연차는 몇 일인가?"
"기술 아키텍처에서 사용하는 데이터베이스는?"
```

#### 3️⃣ 검색 정확도 개선

```
방법 1: 구체적인 질문
"Python에서 데이터를 처리하는 방법" 
→ "Python pandas로 CSV 파일을 읽는 코드"

방법 2: 프로젝트 범위 지정
특정 프로젝트 선택 후 질문

방법 3: 다중 문서 활용
여러 문서를 프로젝트에 추가하여 
상호참조 가능
```

#### 4️⃣ 인덱싱 최적화

```
최적화 팁:
- 큰 문서는 페이지 단위로 분할
- 중복 문서 제거
- 오래된 문서는 비활성화
- 정기적으로 커넥터 동기화 상태 확인
```

## 웹 검색

**인터넷에서 실시간 정보를 검색합니다.**

### 🔍 웹 검색 사용법

#### 기본 사용

```
Chat에서 질문 입력 시:
- "최신" 또는 "오늘의" 를 포함하는 질문
- Onyx가 자동으로 웹 검색 수행

또는 명시적으로:
"웹에서 검색해: 오늘의 날씨"
```

#### 지원하는 검색 엔진

| 검색 엔진 | API 필요 | 특징 |
|----------|---------|------|
| **Google** | Google Custom Search API | 정확도 높음 |
| **Serper** | Serper API | 빠르고 저렴 |
| **Brave** | Brave Search API | 프라이버시 중심 |
| **SearXNG** | 셀프호스팅 | 프라이버시 보호 |

#### 웹 검색 결과

```
AI 답변 + 인용된 웹사이트 링크
→ 출처 클릭으로 원본 페이지 이동 가능
```

## AI 에이전트

**자동으로 복잡한 작업을 수행하는 지능형 봇입니다.**

### 🤖 기본 에이전트

#### 1️⃣ Search Agent

```
기능: 문서 검색 기반으로 정보 찾기
사용: "문서에서 API 키 찾아줘"
```

#### 2️⃣ Web Search Agent

```
기능: 웹에서 정보 검색
사용: "최신 프로그래밍 언어 트렌드를 찾아줘"
```

#### 3️⃣ Research Agent

```
기능: 깊이 있는 연구 수행 (단계별)
사용: "AI 모델의 최신 발전에 대해 심층 보고서 만들어줘"
결과: 다단계 연구 결과 (5-10분 소요 가능)
```

#### 4️⃣ Code Execution Agent

```
기능: 데이터 분석, 계산, 그래프 생성
사용: "이 데이터를 분석하고 그래프 만들어줘"
```

### ⚙️ 에이전트 사용법

```
Chat → "Select Agent" → 에이전트 선택
        ↓
    질문 입력
        ↓
    에이전트가 단계별로 작업 수행
        ↓
    결과 제시
```

## FAQ

### 🤔 자주 묻는 질문

#### Q1: RAG에 업로드된 문서는 어디에 저장되나요?

```
A: 
- PostgreSQL 데이터베이스에 메타데이터 저장
- MinIO/S3에 원본 파일 저장 (설정에 따라)
- 벡터 인덱스는 OpenSearch에 저장

프라이버시:
- 데이터는 자신의 서버에만 저장
- 외부 공유 안 함
```

#### Q2: 얼마나 많은 문서를 업로드할 수 있나요?

```
A:
- 파일 개수: 제한 없음
- 파일 크기: 기본 250MB (설정 가능)
- 전체 저장소: 하드드라이브 크기에 따라

권장:
- 프로젝트당 1,000개 이상의 문서 가능
- 검색 정확도 유지를 위해 관련 문서만 포함
```

#### Q3: RAG 답변의 정확도를 높이려면?

```
A:
1. 문서 품질 향상
   - 명확한 제목과 구조
   - 중복 제거

2. 질문 구체화
   - 구체적인 질문
   - 핵심 단어 포함

3. 프로젝트 분리
   - 주제별로 프로젝트 분류
   - 불필요한 문서 제거

4. 커넥터 활용
   - 자동 동기화로 최신 문서 유지
```

#### Q4: 웹 검색과 RAG를 함께 사용할 수 있나요?

```
A: 네! Onyx는 자동으로 두 가지를 결합합니다.

예시:
"회사 정책과 최신 법규를 고려하면
뭘 해야 하나?"

→ RAG: 회사 정책 검색
→ 웹 검색: 최신 법규 검색
→ AI: 두 정보를 종합하여 답변
```

#### Q5: API Key가 없어도 Onyx를 사용할 수 있나요?

```
A: 제한적으로 가능합니다.

가능한 기능:
✅ RAG (로컬 LLM 사용)
✅ 웹 검색 (SearXNG)
✅ 코드 실행
✅ 음성 모드

필요한 기능:
❌ 최신 AI 모델 사용
❌ 이미지 생성

해결책:
- 로컬 LLM 사용 (Ollama)
- 또는 OpenAI/Anthropic API 키 설정
```

#### Q6: 프라이빗 데이터는 안전한가요?

```
A: 네, 안전합니다.

보안 기능:
- 엔드투엔드 암호화
- 역할 기반 접근 제어 (RBAC)
- 감사 로그 (쿼리 기록)
- 사용자 및 팀 관리
- 데이터는 자신의 서버에만 저장

추가 보안 (Enterprise):
- SSO (Single Sign-On)
- SAML
- SCIM (사용자 프로비저닝)
```

#### Q7: 대용량 문서 처리는 어떻게 하나요?

```
A:

1. 문서 분할
   - 큰 PDF는 페이지 단위로 분할
   - 챕터별로 나누기

2. 배치 업로드
   - 한 번에 여러 파일 업로드
   - 커넥터로 자동 동기화

3. 청킹 설정
   - 문서를 작은 청크로 자동 분할
   - 검색 정확도 향상

권장:
- 한 프로젝트 최대 10,000 페이지
- 더 크면 프로젝트 분리
```
